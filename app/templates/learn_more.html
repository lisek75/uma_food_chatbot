The UMA Food Chatbot, designed for a Japanese restaurant named UMA, uses Dialogflow's Natural Language Processing (NLP) capabilities to assist users in managing their orders seamlessly. The chatbot supports adding or removing items, completing orders, and tracking order status in real-time, making the ordering process efficient and interactive.

📋 Basic Knowledge Requirements

  • Basic Python Programming
  • Familiarity with FastAPI
  • Basic knowledge of MySQL
  • Understand Dialogflow and handle JSON data

🔑 Key Learnings

  • Create and manage intents and entities in Dialogflow using NLP
  • Handle webhook requests from Dialogflow using FastAPI
  • Interact with a MySQL database to manage orders and food items
  • Manage user sessions to ensure data consistency
  • Implement async operations for better performance and handling multiple requests

📖 Story Behind the Code

  • Webhook Processing: Processes requests from Dialogflow, extracts intent, entities, and parameters, and routes to appropriate functions.
  • New Order: Clears session data, lists available items, and guides user to specify food items and quantities.
  • Add/Remove Items: Updates session with specified food items and quantities.
  • Complete Orders: Prompts for confirmation, saves order in database, calculates total price, and clears session data.
  • Track Orders: Retrieves and informs user of order status.
  • Session Management: Cleans up inactive sessions.

🏛️ Architecture

The user journey architecture outlines interactions from greeting to order tracking, detailing actions like starting an order and adding items.


The technical architecture describes processing the interactions via a front-end service, Dialogflow for intent detection, a Webhook Service, and Fast API, ensuring accurate and efficient responses.
