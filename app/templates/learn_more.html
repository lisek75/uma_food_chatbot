The UMA Food Chatbot, designed for a Japanese restaurant named UMA, uses Dialogflow's Natural Language Processing (NLP) capabilities to assist users in managing their orders seamlessly. The chatbot supports adding or removing items, completing orders, and tracking order status in real-time, making the ordering process efficient and interactive.

ðŸ“– Short Story Behind the Code

  • Webhook Handling:
    • The chatbot processes incoming webhook requests from Dialogflow, extracts information (intent, entities, parameters), and routes requests to appropriate functions.
  • New Order:
    • On initiating a new order, the chatbot clears existing session data and provides a list of available items, guiding the user to specify their desired food items and quantities.
  • Add/Remove Items:
    • Users can add or remove items by specifying the food and quantity. The chatbot updates the session based on the recognized items and quantities.
  • Complete Orders:
    • The chatbot prompts for confirmation to complete orders, saves the order in the database, calculates the total price, and clears session data upon confirmation.
  • Track Orders:
    • Users can inquire about order status, and the chatbot retrieves the status from the database and informs the user.
  • Database Helper Functions:
    • Functions ensure efficient data handling, such as checking item existence, saving orders, and calculating prices.
  • Session Management:
    • The application tracks and manages user interactions with active sessions and periodically cleans up inactive sessions for resource efficiency.
